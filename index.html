Metamorphic Malware

January 1, 2019 by Kehinde

This blog post gives an overview of metamorphic malware. It also explains some of its mutation techniques.

Metamorphic malwares are those that are able to change their codes. They do this by hiding their instructions in the host machine's program code. These types of malwares can take on different forms

These metamorphic malwares are able to evade being detected by intrusion detection systems (IDS). This is because the IDS are only trained to recognise specific code versions.

When these IDS encounter new codes, they are often unable to detect them.

Mutation Techniques

The process emlopyed by metamorphic malwares in changing their codes is known as mutation. Several mutation techniques are used by these metamorphic malwares. They include; garbage code insertion, instruction reordering among others.

Garbage Code Insertion

This is one of the most common mutation technique. It involves making the code variant as unrecognisable as possible from the original sample.

NOP and PUSH EAX followed by POP EAX

This could either be "do nothing" or "dead" code. It involves the addition of junk or garbage instruction sequences.

Instruction Reordering

This involves changing the flow of control by manipulating the code's conditional statements.

SUB R1, R2
SUB R3, R4 becomes
SUB R3, R4
SUB R1, R2.

Subroutine Permutation

A program having n independent routines has n! possible variants.

Registry Swapping

This involves the replacement of Variables

Layers of Mutation

December 23, 2018 by Kehinde

There are several layers of attack for metamorphic malwares as listed below;.

  • Network Layer - IP Packet Splitting.
  • Exploit Layer - Polymorphic shellcode.
  • Application Layer - Protocol rounds, SSL null record evasion.

Infection Techiques

December 27, 2018 by Kehinde

There are a number of infection techniques used by metamorphic malware. These techniques will be explained in the following section.

  1. Cavity Insertion - Exploiting unused portions of executables.
  2. Jump-table Manipulation - Exploiting control structures.
  3. Data-segment Expansion - Expanding the host machine's data segment.
